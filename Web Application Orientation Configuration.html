This document specifies a configuration property to describe the preferred view orientations of web applications.

Introduction

A Web Application may only be designed to work in a specific orientation, or it may work in any orientation, but the Web Application author would prefer the application to start using a specific orientation. A Web Application configuration document, for example a Widget configuration document or WebApp Manifest, may use the orientation configuration property to specify their preferred behaviour.

See also [[CSS Device Adaptation]] and CSS Media Queries [[MEDIAQ]]

Definitions

Media feature
The term media feature is defined in [MEDIAQ].
Web application
A Web application is an application built and shipped using Web technology that may be running inside of a browser, as a widget, or in another type of container designed for this purpose.
Widgets
The term widget is defined in [[WIDGETS]].

The orientation configuration property

Name: orientation
Initial value: auto

A list denoting the orientations supported by the Web Application, in order of preferred starting orientation.

This property is used to request that a document is displayed in portrait or landscape mode. For a UA/device where the orientation is changed upon tilting the device, an author can use this property to inhibit the orientation change.

The orientation configuration property accepts the following enumerated values:

'portrait'
the Web Application should be locked to portrait presentation.
'landscape'
the Web Application should be locked to landscape presentation.
'portrait landscape'
the UA should enable both portrait and landscape orientation, but on start the presentation should be in portrait orientation.
'landscape portrait'
the UA should enable both portrait and landscape orientations, but on start the presentation should be in landscape orientation.
'auto'
The UA automatically chooses the orientation based on the device's normal mode of operation. The UA may choose to change the orientation of the presentation when the device is tilted.

Note: portrait, landscape and auto behave in the same fashion as the orientation viewport property as defined by the [[CSS Device Adaptation]] spec.

Examples

An example using the Mozilla webapp manifest format:


  {
    "name": "MozillaBall",
    "description": "Exciting Open Web development action!",
    "default_locale": "en",
    "orientation": "landscape"
  }

An example using [[WIDGETS]]:


<widget xmlns="http://www.w3.org/ns/widgets"
 		id="http://wookie.apache.org/widgets/butterfly"
 		version="0.1"
        width="420"
        height="350"
        orientation="auto"
        >
  <name>Butterfly</name>
  <description>Paint colourful butterflies!</description>
